The error `errno=27 : File too large` indicates that you are trying to extract a file larger than 4 GB onto a file system that does not support it.

Your USB drive is likely formatted as **FAT32**, which has a maximum file size limit of 4 GB. The extracted ISO for *Grand Theft Auto: San Andreas* is a dual-layer DVD image (approximately 4.7 GB or larger), which exceeds this limit.

### Option 2: Split the ISO (Recommended for PS2 via OpenPS2Loader)

1.  Keep your USB as FAT32.
2.  Extract the ISO to your computer's internal drive (which supports large files) first:
    ```bash
    7z x "Grand\ Theft\ Auto\ -\ San\ Andreas\ \(USA\)\ \(v3.00\).7z" -o/home/user/Downloads
    ```
3.  Use a tool to split the ISO. The easiest way on Arch Linux is using `split`:
    ```bash
    sudo mount /dev/sdb1 /mnt/usb
    sudo su
    cd /mnt/usb
    split -b 1024M "/home/user/Downloads/input_file.iso" "output_prefix.iso.part"
    ```
    *(Note: Many users prefer the Windows tool "USBUtil" for this to create the `.iso.0`, `.iso.1` format automatically, but OPL can handle generic splits if named correctly).*
    
    Alternatively, install `opl-ps2-tools` or similar utilities designed for OPL formatting if available in the AUR.

### Summary
*   **FAT32 Limitation:** Max file size is 4GB.
*   **San Andreas ISO:** ~4.7GB (Too big for FAT32).
